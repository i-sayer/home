The following files will invoke the sample creation mode.

  1. wav/aif/aiff - 8 or 16 bit PCM, multi channel ok but only the first channel is used.
  2. bin - 8 bit linear or a-law companded*

*because these ROM image files have no header, there is no way to determine if the data is compressed or not, the system assumes all such files to be a-law compressed unless the file name contains the digits '27'. This was simply expedient due to the nature of the files I had at hand, the uncompressed ones tended to have the ROM code included in the file name (ie. Linn Kick 2716.bin).

All data is normalized for display and by default a 1/2 sample rate copy is selected. The trim slider allows skipping the first (up to) 256 samples. The ratio buttons allow you to select other ratios of decimation.

You can select up to 8 files, each will have it's own variation in the final patch.

Press the Get PCH2 button to create the patch file which you can then download. The patch file will contain information from the file table in it's textpad.
